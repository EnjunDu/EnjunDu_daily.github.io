# 比对修改前后代码块的作用

  1. 对于 test_init_raises（） 函数，其用于验证创建的 Vector2D 对象是否与预期结果 exp 相等。—— 修改前后代码作用相同

  2. 对于 test_repr和test_str 函数：
    修改前：

    其作用为验证 Vector2D 对象的 '__repr__' 方法的输出。通过建立字典 capture_stdout 来捕获 capture_stdout。print 函数的目的为将 Vector2D(1.0, 2.0) 对象的 __repr__ 方法的输出打印到标准输出。以便实际使用时开发者可以看到程序输出结果，帮助理解程序执行的结果和确保 __repr__ 方法的实现符合预期。 然后使用 assert capture_stdout["stdout"] == "vector.Vector2D(1.0, 2.0)\n " 来验证捕获的输出与预期输出是否相符。
    gpt3.5 补充后：

    缺点：

    • 功能减少：一般来说为了符合开发者调试，test 时需要 print 函数以便开发者了解程序的输出，gpt 填充的 assert 里忽略了 print，直接对比返回值，其只适用于 repr 和 str 函数的输出场景
    • 调试困难：如果我们 assert 报错后，最初的代码可以通过 print 来留下更多的调试信息，而 gpt 生成的返回值无法做到这一步，需要额外增加 print 代码来了解报错原因

    改进：当我们给 gpt 输入：“这个 test case 缺少 assert 语句 请补全，要求调试的过程中方便开发者直观看到输出过程” 后，代码输出增加了 print 语句，优化成功。

  3. test_add 和 test_sub 和 test_mul_float``test_div 和 'test_abs' 和 'test_less_than' 函数：验证两个 Vector2D 对象相加 / 减 / 乘 / 除 / 取绝对值 / 是否小于的结果是否与预期结果相等。gpt 改后与之前效果相同。

# 分析一：断言在验证的东西

  1. assert exp == Vector2D(x, y)
    • 使用参数化设置，传入不同的 x 和 y 值，以及期望的 Vector2D (x,y) 对象 'exp'
    • 测试通过创建一个新的 Vector2D 对象并将其与预期的对象进行比较来验证初始化是否正确
  2. assert capture_stdout["stdout"] == "vector.Vector2D(1.0, 2.0)\n"
    • 验证 repr 方法返回的 Vector2D 对象是否用正确字符串表示
  3. assert capture_stdout["stdout"] == "(1.0, 2.0)\n"
    • 验证 str 方法返回的 Vector2D 对象是否用正确字符串表示
  4. assert lhs + rhs == exp_res
    • 验证两个 Vector2D 对象相加的结果是否正确。
  5. assert lhs - rhs == exp_res
    • 验证两个 Vector2D 对象相减的结果是否正确。
  6. assert lhs * rhs == exp_res
    • 验证两个 Vector2D 对象相乘（点积）的结果是否正确。
  7. assert lhs * rhs == exp_res —— test_mul_float
    • 验证 Vector2D 对象与浮点数相乘的结果是否正确。
  8. assert lhs / rhs == exp_res
    • 证 Vector2D 对象与浮点数相除的结果是否正确。
  9. assert abs(rhs) == lhs
    • 验证 Vector2D 对象的绝对值（模）是否正确。
  10. assert lhs != rhs
    • 验证当 Vector2D 对象与不同类对象进行比较时，是否返回 False
  11. assert rhs < lhs
    • 验证 Vector2D 对象的 “小于” 比较运算是否正确。

# 分析二：断言是否依赖于具体的 SUT (被测软件) 的输出及原因

  1. assert exp == Vector2D(x, y)
    • 依赖
    • 这个测试通过创建 Vector2D 对象来验证其初始化功能
    • 断言 exp == Vector2D(x, y) 依赖于 Vector2D 的 __eq__ 方法是否正确实现。
    • 如果 Vector2D 类的初始化或 __eq__ 方法有问题，这个断言将无法通过。因此，断言依赖于 Vector2D 类的具体实现（即 SUT 的输出）。
  2. assert capture_stdout["stdout"] == "vector.Vector2D(1.0, 2.0)\n"
    • 依赖
    • 断言 capture_stdout["stdout"] == "vector.Vector2D(1.0, 2.0)\n" 依赖于 __repr__ 方法的具体输出。
    • 也依赖 Vector2D (1.0,2.0) 的具体输出
  3. assert capture_stdout["stdout"] == "(1.0, 2.0)\n"
    • 依赖
    • 它依赖于 Vector2D 类中 __str__ 方法的具体实现。
  4. assert lhs + rhs == exp_res
    • 依赖
    • 依赖于 Vector2D 类中 __add__ 方法的具体实现。
  5. assert lhs - rhs == exp_res
    • 依赖
    • 依赖于 Vector2D 类中 __sub__ 方法的具体实现。
  6. assert lhs * rhs == exp_res
    • 依赖
    • 它依赖于 Vector2D 类中 __mul__ 方法对另一个 Vector2D 对象的具体实现。
  7. assert lhs * rhs == exp_res —— test_mul_float
    • 依赖
    • 依赖于 Vector2D 类中 __mul__ 方法对浮点数的具体实现。
  8. assert lhs / rhs == exp_res
    • 依赖
    • 依赖于 Vector2D 类中 __truediv__ 方法的具体实现
  9. assert abs(rhs) == lhs
    • 它依赖于 Vector2D 类中 __abs__ 方法的具体实现。
  10. assert lhs != rhs
    • 它依赖于 Vector2D 类中 __eq__ 方法的具体实现。
  11. assert rhs < lhs
    • 它依赖于 Vector2D 类中 __lt__ 方法的具体实现。

# 分析三：断言为何能生成

  1. assert exp == Vector2D(x, y)
    • 函数名：投喂给 LLM 的函数名为 test_from_values ，即从数值上进行比较，而函数中对 x,y 进行初始化，并且定义了一个 exp=Vector2D，故 LLM 可以判断断言为判断 exp 是否和 Vector2D (x,y) 相等
  2. assert capture_stdout["stdout"] == "vector.Vector2D(1.0, 2.0)\n"
    • 函数名，投喂给 LLM 的函数名为 test_repr ，即为测试该 Vector2D 的标准字符串表示
    • 在代码里出现了 capture_stdout，即为捕获标准输出。
    • 函数里出现 print(repr(Vector2D(1.0, 2.0))) ，即可知道需要对于标准输出和 repr(Vector2D(1.0, 2.0)) 进行对比
    • 故 LLM 可以依此来判定断言
  3. assert capture_stdout["stdout"] == "(1.0, 2.0)\n"
    • 函数名，投喂给 LLM 的函数名为 test_str ，即为测试该 Vector2D 的字符串表示
    • 在代码里出现了 capture_stdout，即为捕获标准输出。
    • 函数里出现 print(repr(Vector2D(1.0, 2.0))) ，即可知道需要对于标准输出和 str(Vector2D(1.0, 2.0)) 进行对比
    • 故 LLM 可以依此来判定断言
  4. assert lhs + rhs == exp_res
    • 函数名为 test_add , 即为测试 add 方法
    • 函数中出现了 lhs,rhs 和 exp_res
    • 故 LLM 可以判定方法为测试相加是否相等
  5. assert lhs - rhs == exp_res
    • 函数名为 test_sub , 即为测试 add 方法
    • 函数中出现了 lhs,rhs 和 exp_res
    • 故 LLM 可以判定方法为测试相减是否相等
  6. assert lhs * rhs == exp_res
    • 函数名为 test_mul_vec，即为测试相乘是否结果一致
    • 出现了 lhs,rhs,exp_res 的初始化定义
    • 故 LLM 可以判定方法为测试相乘是否相等
  7. assert lhs * rhs == exp_res —— test_mul_float
    • 函数名为 test_mul_vec，即为测试相乘是否结果一致
    • 出现了 lhs,rhs,exp_res 的初始化定义
    • 故 LLM 可以判定方法为测试 Vector2D 与浮点数相乘的结果
  8. assert lhs / rhs == exp_res
    • 函数名为 test_div ，即测试除法
    • 函数中对 lhs,rhs,exp_res 进行初始化
    • 即 LLM 可以判定为测试除法是否正确
  9. assert abs(rhs) == lhs
    • 函数名为 test_abs ，即测试绝对值
    • 函数中对 reh 和 lhs 进行初始化
    • 故可以判定方法为测试绝对值
  10. assert lhs != rhs
    • 函数名为 test_equality_other_class
    • 函数中定义了 lhs: Vector2D 和 rhs: object
    • 故 LLM 可知方法为测试两个不同类进行比较是否返回 False
  11. assert rhs < lhs
    • 函数名为 test_less_than
    • 初始化定义了两个 Vector2D 类 lhs 和 rhs
    • 故 LLM 可知该断言应该为判断对象的小于比较是否正确
